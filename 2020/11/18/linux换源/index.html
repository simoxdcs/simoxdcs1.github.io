## 第一章

因为我 Linux 也是才开始用。。结果自己就遇到了很多麻烦的事情。。就自己在网上东扒西找的姑且算是解决了。。就自己写个东西一面自己在遇到的时候又不会了。。
换源：
sudo pacman-mirrors -i -c China -m rank 更新镜像
sudo vim /etc/pacman.d/mirrorlist 查看选择的源
sudo vim /etc/pacman.conf 设置源（可以自己添加源链接进去）

我用的是 manjaro 这个 archlinux 的衍生版本。。还算是怪好用的
更新 ：
1.sudo pacman -S cuda 安装
2.pacman -Syu 更新系统
3.pacman -Syy 强制更新
4.pacman -Ss keyword 在仓库中搜索含关键字的包（常用）
5.pacman -Qs keyword 搜索已安装的包（常用）
6.pacman -Qi package_name 查询本地安装包的详细信息
7.pacman -Ql package_name 列出该包的文件
8.pacman -Fs keyword 按文件名查找软件库
9.pacman -Si package_name 显示远程软件包的详尽的信息
10.pacman -Qii package_name 使用两个 -i 将同时显示备份文件和修改状态
11.pacman -Ql package_name 要获取已安装软件包所包含文件的列表
12.pacman -Fl package_name 查询远程库中软件包包含的文件
13.pacman -Qk package_name 检查软件包安装的文件是否都存在
14.pacman -Fo /path/to/file_name 查询文件属于远程数据库中的哪个软件包
15.pacman -Qdt 要罗列所有不再作为依赖的软件包 (孤立 orphans)
16.pacman -Qet 要罗列所有明确安装而且不被其它包依赖的软件包
17.pactree package_name 要显示软件包的依赖树
18.whoneeds package_name 检查一个安装的软件包被那些包依赖
19.pactree -r package_name 检查一个安装的软件包被那些包依赖
20…pacman -Sw package_name 只下载包，不安装。
21.pacman -Sc 清理未安装的包文件（常用）包文件位于 /var/cache/pacman/pkg/ 目录
22.pacman -Scc 清理所有的缓存文件（常用）
23.pacman -R package_name 该命令将只删除包，保留其全部已经安装的依赖关系
24.pacman -Rs package_name 在删除包的同时，删除其所有没有被其他已安装软件包使用的依赖关系（常用）
25.pacman -Rsc package_name 在删除包的同时，删除所有依赖这个软件包的程序
26.pacman -Rd package_name 在删除包时不检查依赖
